The ToolRegistry is the single source of truth for all tool schemas and handlers.

How tools work:
- Tools are registered at module-load time via registry.register(name, toolset, schema, handler, ...).
- Discovery is automatic: abstract/tools/discover.py scans .py files for module-level registry.register() calls using AST parsing, then imports them.
- The agent loop calls registry.get_definitions(tool_names) to get OpenAI-format schemas for the LLM.
- Tool dispatch goes through registry.dispatch(name, args), which handles both sync and async handlers.
- check_fn determines availability — results are TTL-cached for 30 seconds.

Built-in tool conventions:
- Tool names use snake_case.
- Each tool belongs to a toolset (e.g. "filesystem", "code").
- Handlers accept a single dict argument and return a JSON string.
- Errors are returned as {"error": "message"} via tool_error() helper.
- Success results use tool_result() helper.

Cron tool conventions:
- Scheduled tasks are background one-shot tasks, not a batch queue or an automatic loop mechanism.
- Periodic work, polling, or long-running observation must be expressed as recursive scheduling: wait for the task result, then schedule the next one-shot task.
- Raw scheduled-task output is internal Agent input and log content, not a user message; if the user needs the result, actively summarize it.

Sub-agent conventions:
- After launching a sub-agent with run_subagent, you MUST wait for the [subagent-result] message before interacting with it again.
- Do NOT call chat_subagent to check whether the sub-agent is alive, to ask "are you there", or to urge it to respond faster. The system will reject such calls while the sub-agent is still generating its current response.
- Use chat_subagent only to reply to the sub-agent's questions, add missing context, or correct its direction AFTER you have received its latest response.
- Do NOT call chat_subagent just to send the initial prompt; put all initial context into the initial_prompt parameter of run_subagent.
- Within the same parent session, only one instance of a given sub-agent name can be active or queued at a time. If you need to restart a sub-agent, stop it first with stop_subagent.

Tool selection rules:
- Always prefer dedicated tools over shell workarounds.
- For reading files, use ``read_file``; for editing, use ``edit_file``; for create, use ``write_file``.
- For searching code, use ``search_files`` or ``grep``.
- Never use ``run_command`` to invoke cat, type, head, tail, find, grep, or similar file utilities.
- Never use ``run_python`` to write files; always use ``write_file`` or ``edit_file`` instead.
