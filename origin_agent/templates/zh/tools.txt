ToolRegistry 是所有工具模式和处理器（handler）的唯一数据源。

工具运作方式：
- 工具通过 registry.register(name, toolset, schema, handler, ...) 在模块加载时注册。
- 自动发现：abstract/tools/discover.py 使用 AST 解析扫描 .py 文件，查找模块级别的 registry.register() 调用，自动导入。
- Agent 循环调用 registry.get_definitions(tool_names) 获取 OpenAI 格式的工具定义，供 LLM 使用。
- 工具分发通过 registry.dispatch(name, args)，同时支持同步和异步处理器。
- check_fn 决定工具可用性，结果有 30 秒 TTL 缓存。

内置工具约定：
- 工具名使用 snake_case。
- 每个工具属于一个 toolset（如 "filesystem"、"code"）。
- 处理器接收一个字典参数，返回 JSON 字符串。
- 错误通过 tool_error() 辅助函数返回 {"error": "message"}。
- 成功结果使用 tool_result() 辅助函数。

定时任务工具约定：
- 定时任务是后台一次性任务，不是批量排队工具，也不是自动循环工具。
- 周期、轮询或长期观察应通过“任务完成后收到结果，再设置下一次任务”的递归调度表达。
- 定时任务的原始输出是 Agent 内部输入和日志内容，不是用户消息；需要告知用户时必须主动总结。
