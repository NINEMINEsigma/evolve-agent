# 模式1：研究 [MODE: RESEARCH]

> **前置阅读**：本 skill 是 RIPER-5 协议的阶段1。公共协议规则（模式声明要求、核心思维原则、关键协议指南、代码处理指南、模式转换信号、任务文件模板、占位符定义等）定义在 `riper-core/SKILL.md`。在使用本 skill 前，必须先读取该文件以获取公共约束。

## 目的

信息收集和深入理解。

## 核心思维应用

- 系统地分解技术组件
- 清晰地映射已知/未知元素
- 考虑更广泛的架构影响
- 识别关键技术约束和要求

## 允许

- 阅读文件
- 提出澄清问题
- 理解代码结构
- 分析系统架构
- 识别技术债务或约束
- 创建任务文件（参见 `riper-core` 中的任务文件模板）—— 必须经用户明确同意后方可创建
- 创建功能分支 —— 必须经用户明确同意后方可创建

## 禁止

- 建议
- 实施
- 规划
- 任何行动或解决方案的暗示

## 研究协议步骤

1. 创建功能分支（必须提问，经用户同意后方可创建；禁止未经询问直接创建）：

   ```java
   git checkout -b task/[TASK_IDENTIFIER]_[TASK_DATE_AND_NUMBER]
   ```

2. 创建任务文件（必须提问，经用户同意后方可创建；禁止未经询问直接创建）：

   你必须通过调用指令（如 `Get-Date`）获取当前的时间，因为你的知识库中的时间是冻结的。

   ```java
   mkdir -p .tasks && touch ".tasks/${TASK_FILE_NAME}_[TASK_IDENTIFIER].md"
   ```

3. 分析与任务相关的代码：
   - 识别核心文件/功能
   - 追踪代码流程
   - 记录发现以供以后使用
