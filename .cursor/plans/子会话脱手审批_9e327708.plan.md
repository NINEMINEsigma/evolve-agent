## 问题

子会话（SubAgentLoop）的非 readonly 工具审批完全绕过了 `request_user_confirm`，直接走 `pending_approvals` → `orchestrator._collect_and_inject` → 父 Agent 的 `approval_subagent` 链路。即使父 Agent 在脱手模式，子会话的审批仍然交给主 Agent 处理。

## 方案

**统一审批入口：让子会话的审批逻辑复用 `request_user_confirm`，由它自动判断脱手/非脱手。**

### 改动点 1：`origin_agent/subagent/orchestrator.py`（`_start_subagent`）

在启动子会话后，若父 Agent 处于脱手模式，将子会话的 `session_id` 也注册到脱手模式：

```python
from component.approval import set_handsfree_mode, is_handsfree_mode

# 在 loop = SubAgentLoop(...) 之后，_push_subagent_ws 之前
if self._parent_session_id and is_handsfree_mode(self._parent_session_id):
    set_handsfree_mode(session_id, True)
```

这样 `request_user_confirm` 检查 `is_handsfree_mode(session_id)` 时，子会话会自动进入脱手审批路径。

### 改动点 2：`origin_agent/subagent/loop.py`（`_queue_for_approval`）

在 `_queue_for_approval` 开头增加脱手模式判断：

```python
from component.approval import is_handsfree_mode, request_user_confirm

async def _queue_for_approval(self, tc: ToolCall) -> dict[str, Any]:
    if is_handsfree_mode(self._session_id):
        # 脱手模式：直接走审批模型
        result = await request_user_confirm(
            session_id=self._session_id,
            tool_name=tc.name,
            args=dict(tc.arguments) if tc.arguments else {},
            reason="Sub-agent initiated tool call",
            content=f"Sub-agent {tc.name} tool call",
            ask_agent_callback=None,  # 不启用 dialog 模式
        )
        if result.action in ("allow_once", "allow_always"):
            self._emit("approval_decision", tool_call_id=tc.id, tool_name=tc.name, content="approved")
            return await self._execute_approved_tool(tc)
        self._emit("approval_decision", tool_call_id=tc.id, tool_name=tc.name, content=f"rejected: {result.deny_reason}")
        return self._make_tool_msg(tc.id, f"Tool call denied: {result.deny_reason}")

    # 原有逻辑：非脱手模式入 pending_approvals，阻塞等待父 Agent
    ...
```

### 关键点

- **非脱手模式**：子会话的 `session_id` 未注册脱手模式，`_queue_for_approval` 保持原逻辑，继续走 `pending_approvals` → 父 Agent `approval_subagent` 链路。子会话无 WebSocket 连接，不会误走前端审批。
- **脱手模式**：子会话的 `session_id` 被注册到脱手模式，`_queue_for_approval` 直接调用 `request_user_confirm`，自动走 `_handsfree_confirm`（审批模型），不经过主 Agent。
- **审批后**：无论允许还是拒绝，都通过 `_emit` 推送 `approval_decision` 事件到前端面板，和原有行为一致。
