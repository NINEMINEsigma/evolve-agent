# 硬编码字面量提取计划

基于全量筛查结果，将以下硬编码值逐项提取到 `entity/constant.py` 或引用已有 `RuntimeContext` 字段。每项独立，用户可逐个跳过。

---

## 待改项列表（按优先级排序）

### T1 — 超时值统一（`120` 秒，10+ 文件）

当前 `120` 秒超时散布在工具调用、子进程、审批加载、前端构建等场景，无统一来源。

涉及文件：
- `origin_agent/abstract/mcp/client.py` — 已有 `_DEFAULT_TOOL_TIMEOUT = 120`，其余位置引用它
- `origin_agent/component/tools/frontend.py:79,96,113,130` — 前端 install/build 超时
- `origin_agent/component/extools/pip.py:46,48` — pip install 超时
- `origin_agent/component/extools/ssh_tools.py:162,166,230,234` — SCP 上传/下载超时
- `origin_agent/component/extools/mermaid_tools.py:86,90` — Mermaid 加载超时
- `origin_agent/component/extools/excalidraw_render.py:120,124,138,143` — Excalidraw 渲染超时
- `origin_agent/component/approval.py:181,187,188,364,371` — 审批模型加载 + 审批等待超时
- `origin_agent/gateway/server.py:669` — LLM 请求超时

方案：在 `entity/constant.py` 新增分类超时常量（如 `SUBPROCESS_TIMEOUT_DEFAULT`、`APPROVAL_MODEL_LOAD_TIMEOUT`、`APPROVAL_WAIT_TIMEOUT`、`FRONTEND_BUILD_TIMEOUT` 等），各文件引用。

---

### T2 — 超时值统一（`300` 秒，5+ 文件）

涉及文件：
- `origin_agent/component/extools/ffmpeg_tools.py:72` — ffmpeg 默认超时
- `origin_agent/component/extools/cron_tools.py:493,512` — cron 任务执行超时
- `origin_agent/abstract/mcp/client.py:1464,1506` — SSE read timeout
- `origin_agent/abstract/mcp/oauth.py:473` — OAuth 超时
- `origin_agent/abstract/mcp/oauth_manager.py:445` — OAuth 配置默认超时

方案：提取为 `CRON_TASK_TIMEOUT`、`FFMPEG_DEFAULT_TIMEOUT`、`SSE_READ_TIMEOUT` 等。

---

### T3 — 工具默认超时 `30` 秒未引用 ctx

`RuntimeContext.tool_timeout = 30` 已存在，但以下位置各自硬编码 `30`，不引用 ctx：
- `origin_agent/system/sandbox.py:234` — 默认参数
- `origin_agent/component/tools/shell.py:70` — 硬编码 `timeout=30`
- `origin_agent/component/tools/skills.py:262` — 硬编码 `timeout=30`

方案：这些位置改为从 `get_runtime_context().tool_timeout` 获取。

---

### T9 — 命名空间前缀不一致

`shell.py:58` 和 `background_service.py:97` 的前缀列表 `("ws:", "fork:", "fix:")` 遗漏了 `"skills:"`，与 `sandbox.py:269` 的 `("ws:", "fork:", "fix:", "skills:")` 不一致。

方案：在 `entity/constant.py` 定义 `NAMESPACE_PREFIXES` 列表，三处统一引用。

---

### T10 — LLM 默认值二次硬编码

- `origin_agent/component/llm.py:102` — `"https://api.openai.com/v1"`
- `origin_agent/component/llm.py:104` — `"gpt-4o"`

方案：ctx 已保证有默认值，`llm.py` 中的 `or` 兜底可移除或改为断言。

---

### T11 — User-Agent 字符串重复

- `origin_agent/component/extools/web_search.py:29,31`
- `origin_agent/component/extools/web_fetch.py:19,21`

方案：提取到 `entity/constant.py` 或共享 utils 模块。

---

### T12 — Gateway 端口/地址在 server.py 和 vite.config.ts 中二次硬编码

- `origin_agent/gateway/server.py:1163` — `host="127.0.0.1", port=8765` 默认参数
- `origin_agent/frontend/vite.config.ts:14` — `ws://127.0.0.1:8765`

方案：`server.py` 改为从 ctx 获取；`vite.config.ts` 是构建时配置，可保留硬编码或用 env 变量。

---

### T13 — `4096` 多语义重复

6 个位置使用 `4096`，语义不同：
- `system/context.py:68` — LLM 输出 token 上限
- `system/context.py:82` — 审批模型上下文窗口
- `abstract/mcp/client.py:43,706` — MCP token cap
- `component/approval.py:105` — 审批模型 n_ctx fallback
- `__main__.py:120` — CLI fallback
- `component/tools/filesystem.py:620` — 文件采样字节（语义完全不同）

方案：前 5 处已通过 ctx 传递，`approval.py:105` 和 `__main__.py:120` 的 fallback 可引用 ctx 默认值。`filesystem.py:620` 的 `4096` 是文件嗅探字节数，独立保留或命名。

---

### T14 — 会话角色字符串（`"user"` / `"assistant"` / `"system"` / `"tool"`）

30+ 处直接使用字符串字面量。`gateway/chat.py:34` 已定义 `SYSTEM = "system"` 但仅局部使用。

方案：在 `entity/constant.py` 定义 `Role` 枚举或常量集，全局引用。改动面大，可跳过。

---

### T15 — 审批模型端口 `8081` 在 `__main__.py` 二次硬编码

- `origin_agent/__main__.py:122` — `int(cli.get("approval_model_port", 8081))`

方案：引用 `RuntimeContext` 的字段默认值或 `entity/constant.py`。

### T16 — Windows 进程创建标志重复

`subprocess.CREATE_NEW_PROCESS_GROUP` 在 3 处硬编码：
- `origin_agent/system/sandbox.py:294`
- `origin_agent/component/extools/cron_tools.py:491`
- `origin_agent/component/extools/background_service.py:116`

方案：提取为 `system/subprocess_utils.py` 中的辅助函数，已有该模块适合放置。

---

### T17 — 路径片段硬编码

- `"workspace/agentspace/uploads"` — `gateway/server.py:686,808` 两处重复
- `"workspace/logs/memory"` — `memory/provider.py:38` fallback

方案：提取为常量。diff_tools 中的目录名也可考虑从 RuntimeContext 获取。

方案：建议跳过。
