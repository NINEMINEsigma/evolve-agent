## 核心设计决策

### 分叉 = `parent` 字段，与 `continuation` 无关

- `continuation` 是单字符串，只记录**自动旋转**产生的主延续线（向后兼容）
- **分叉**通过多次调用 `create_with_context(summary, same_parent_sid)` 实现
- 每次调用产生一个新会话条目，其 `parent` 指向同一父会话
- 前端构建树时完全基于 `parent` 字段，忽略 `continuation`

### 收藏区 = `pinned` 会话的快捷入口

- 收藏区显示在侧边栏顶部，是 `pinned` 会话的**扁平列表**
- 点击收藏项 = 切换到对应会话（不强制展开树）
- 下方树形结构**保留所有会话**（含 pinned），不抽离

## 后端变更

### 1. `entry/agent.py` — 新增 `branch_session()`

```python
async def branch_session(self, session_id: str) -> dict:
    """从任意会话创建分支延续（手动分叉）。
    
    与自动旋转 _rotate_session 的区别：
    - 不设置旧会话的 continuation 字段
    - 只通过 parent 字段建立血统关系
    """
    if self._session_manager is None:
        return {"error": "session manager not ready"}
    
    # 加载历史（归档会话自动从磁盘加载）
    history: List[Dict[str, Any]] = self._get_history(session_id)
    
    # 检查是否已有压缩摘要
    summary: str = ""
    if history and history[0].get("role") == "system":
        content: str = history[0].get("content", "")
        if "[Context Summary]" in content or "[Session continuation summary]" in content:
            summary = content.split("\n", 1)[1] if "\n" in content else content
    
    # 无摘要则生成
    if not summary and history:
        self._histories[session_id] = history
        self._last_prompt_tokens[session_id] = 0
        summary = await self._compress_history(session_id) or "(Session branched)"
    
    if not summary:
        summary = "(Session branched)"
    
    # 创建新会话，parent 指向源会话（不设置 continuation）
    new_sid: str = self._session_manager.create_with_context(summary, session_id)
    
    # 新会话历史加载到内存
    self._histories[new_sid] = self._load_history_from_disk(new_sid)
    self._last_prompt_tokens[new_sid] = 0
    
    return {"session_id": new_sid, "parent": session_id}
```

#### 4.3 树形数据结构（基于 `parent`，忽略 `continuation`）

```ts
interface SessionTreeNode {
  session: SessionInfo;
  children: SessionTreeNode[];
  subtreeLastActivity: number;
}

function buildSessionTree(sessions: SessionInfo[]): SessionTreeNode[] {
  const childrenMap = new Map<string, SessionInfo[]>();
  
  for (const s of sessions) {
    if (s.parent) {
      if (!childrenMap.has(s.parent)) childrenMap.set(s.parent, []);
      childrenMap.get(s.parent)!.push(s);
    }
  }
  
  function buildNode(session: SessionInfo): SessionTreeNode {
    const children = (childrenMap.get(session.id) || [])
      .map(buildNode)
      .sort((a, b) => b.subtreeLastActivity - a.subtreeLastActivity);
    const subtreeLastActivity = Math.max(
      session.last_activity_at || session.created_at,
      ...children.map((c) => c.subtreeLastActivity)
    );
    return { session, children, subtreeLastActivity };
  }
  
  // 根节点 = 无 parent 的会话
  const roots = sessions
    .filter((s) => !s.parent)
    .map(buildNode)
    .sort((a, b) => b.subtreeLastActivity - a.subtreeLastActivity);
  
  return roots;
}
```

#### 4.6 递归树节点组件

```tsx
function SessionTreeItem({
  node,
  depth = 0,
}: {
  node: SessionTreeNode;
  depth?: number;
}) {
  const [expanded, setExpanded] = useState(true);
  const s = node.session;
  const hasChildren = node.children.length > 0;
  const isArchived = s.status === "archived";
  
  return (
    <div>
      <div
        className={`session-item ${s.id === sessionId ? "active" : ""} ${isArchived ? "archived" : ""}`}
        style={{ paddingLeft: `${12 + depth * 16}px` }}
        onClick={() => switchSession(s.id)}
        onContextMenu={(e) => handleContextMenu(e, s.id)}
      >
        <div className="session-item-content">
          {hasChildren && (
            <span className="tree-toggle" onClick={(e) => { e.stopPropagation(); setExpanded(!expanded); }}>
              {expanded ? "▼" : "▶"}
            </span>
          )}
          {!hasChildren && <span className="tree-spacer" />}
          
          <div className="session-item-title">
            {s.pinned && <span className="pin-badge">★</span>}
            {s.title || s.id.slice(0, 8) + "..."}
            {isArchived && <span className="archived-badge">已归档</span>}
          </div>
          
          {/* 归档会话显示"继续"按钮 */}
          {isArchived && s.id === sessionId && (
            <button className="branch-btn" onClick={(e) => { e.stopPropagation(); branchSession(s.id); }}>
              继续
            </button>
          )}
        </div>
        <div className="session-item-sub">
          <span className="session-item-id">{s.id}</span>
          <span className="session-item-time">{formatTime(s.last_activity_at || s.created_at)}</span>
        </div>
      </div>
      
      {expanded && node.children.map((child) => (
        <SessionTreeItem key={child.session.id} node={child} depth={depth + 1} />
      ))}
    </div>
  );
}
```

**注意：** `SessionTreeItem` 作为 App 组件内部函数，可以访问闭包变量。实际实现时确保它在 `App()` 内部定义。

## 数据流示意

```mermaid
graph TD
    A[用户点击归档会话的"继续"] --> B[前端 POST /api/sessions/{sid}/branch]
    B --> C[AgentLoop.branch_session]
    C --> D[_get_history 加载磁盘历史]
    D --> E{已有压缩摘要?}
    E -->|是| F[提取摘要]
    E -->|否| G[_compress_history 生成摘要]
    F --> H[SessionManager.create_with_context<br/>parent = sid, 不设置 continuation]
    G --> H
    H --> I[创建新会话 JSONL 含摘要 system 消息]
    I --> J[返回 {session_id, parent}]
    J --> K[前端 switchSession 切换到新会话]
    K --> L[fetchSessions 刷新列表]
    L --> M[buildSessionTree 重新构建树]
    M --> N[新分支出现在父节点下方]
```

---

## 树形排序规则

1. 根节点按**子树最新活动时间**降序排列
2. 同一父节点下的子节点也按子树最新活动时间降序排列
3. `pinned` 会话同时出现在收藏区和树中，不影响树的排序逻辑

- `parent` 和 `continuation` 已存在于后端返回数据中
- 归档会话历史已持久化到磁盘 JSONL，`_get_history` 自动加载

- 手动分支不设置 `continuation`，自动旋转仍设置 `continuation`（向后兼容）
