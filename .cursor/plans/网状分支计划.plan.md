## 核心设计变更

### 合并摘要：阈值 5w 字符 + 作为 user message

- 拼接阈值从 2000 提升到 **50000 字符**
- 新会话的第一条消息不再是 `role: "system"`，而是 `role: "user"`
- 内容自带说明前缀，让 LLM 识别为继承上下文而非新问题
- 超过 5w 时回退到截断拼接（不调用 LLM 二次合并，避免token爆炸）

### 数据模型：`parent` 升级为 `parents` 数组

- `_index.json` 中 `parent` 升级为 `parents: string[]`
- 现有数据兼容：`parent` 存在时映射为 `parents: [parent]`
- `continuation` 保持单字符串（向后兼容）

### 1. `gateway/chat.py` — `create_with_context` 支持自定义角色

```python
def create_with_context(
    self,
    content: str,
    parent_sid: str | None = None,
    parents: list[str] | None = None,
    role: str = "system",
) -> str:
    """创建新会话并以指定角色的消息作为初始内容。支持多父合并。"""
    new_sid: str = self.create(parent_sid=parent_sid, parents=parents)
    sdir: Path | None = self._store_dir / new_sid if self._store_dir else None
    if sdir:
        sdir.mkdir(parents=True, exist_ok=True)
        msg_path: Path = sdir / "messages.jsonl"
        entry: dict = {"role": role, "content": content}
        try:
            with open(msg_path, "a", encoding="utf-8") as f:
                f.write(json.dumps(entry, ensure_ascii=False) + "\n")
        except Exception as exc:
            logger.warning("Failed to write context for session %s: %s", new_sid, exc)
    
    # 更新主父会话的 continuation
    primary_parent: str | None = (parents[0] if parents else parent_sid) if (parents or parent_sid) else None
    if primary_parent and primary_parent in self._sessions:
        self._sessions[primary_parent]["continuation"] = new_sid
    
    logger.info("Session created | new=%s parents=%s role=%s", new_sid, parents or [parent_sid], role)
    return new_sid
```

### 2. `entry/agent.py` — `merge_sessions()` 重写

```python
async def merge_sessions(self, source_session_ids: list[str]) -> dict:
    """从多个会话创建合并延续。摘要阈值 5w，作为 user message 写入。"""
    if self._session_manager is None:
        return {"error": "session manager not ready"}
    if len(source_session_ids) < 1:
        return {"error": "at least one source session required"}
    if len(source_session_ids) == 1:
        return await self._branch_single(source_session_ids[0])
    
    # 收集各源摘要
    summaries: list[tuple[str, str, str]] = []  # (sid, title, summary_text)
    
    for sid in source_session_ids:
        history: List[Dict[str, Any]] = self._get_history(sid)
        title: str = ""
        if self._session_manager is not None:
            info = self._session_manager.get(sid)
            title = info.get("title", "") if info else ""
        
        summary: str = ""
        if history and history[0].get("role") == "system":
            content: str = history[0].get("content", "")
            if "[Context Summary]" in content or "[Session continuation summary]" in content:
                summary = content.split("\n", 1)[1] if "\n" in content else content
        
        if not summary and history:
            self._histories[sid] = history
            self._last_prompt_tokens[sid] = 0
            summary = await self._compress_history(sid) or f"(Session {sid[:8]})"
        
        if not summary:
            summary = f"(Session {sid[:8]})"
        
        summaries.append((sid, title, summary))
    
    # 拼接内容（阈值 5w 字符，几乎总是直接拼接）
    CONCAT_THRESHOLD: int = 50000
    
    parts: list[str] = [
        "[The following is inherited context from previous sessions. "
        "It is background information, not a new question.]"
    ]
    for sid, title, text in summaries:
        display: str = title.strip() if title.strip() else sid[:8]
        parts.append(f"\n--- From session {sid[:8]}: {display} ---\n{text}")
    
    merged_content: str = "\n".join(parts)
    
    # 超过阈值时截断（不调用 LLM，避免 token 爆炸）
    if len(merged_content) > CONCAT_THRESHOLD:
        merged_content = merged_content[:CONCAT_THRESHOLD] + "\n\n[...truncated due to length limit]"
    
    # 创建新会话，作为 user message 写入
    new_sid: str = self._session_manager.create_with_context(
        merged_content,
        parents=source_session_ids,
        role="user",
    )
    
    # 加载新会话历史到内存
    self._histories[new_sid] = self._load_history_from_disk(new_sid)
    self._last_prompt_tokens[new_sid] = 0
    
    return {
        "session_id": new_sid,
        "parents": source_session_ids,
    }

async def _branch_single(self, session_id: str) -> dict:
    """单源分支（退化为 merge_sessions 的单个源场景）。"""
    result: dict = await self.merge_sessions([session_id])
    return result
```

@app.post("/api/sessions/merge")
async def merge_sessions_endpoint(req: Request):
    body: dict = {}
    try:
        body = await req.json()
    except Exception:
        pass

## user message 结构示例

新会话 `messages.jsonl` 的第一条内容：

```
{"role": "user", "content": "[The following is inherited context from previous sessions. It is background information, not a new question.]\n\n--- From session abc123: 项目依赖安装指南 ---\n我们在项目中安装了 Python 3.10 和 pydantic...\n\n--- From session def456: RenderDoc 源码改造 ---\n已完成 renderdoc 源码下载，下一步准备添加..."}
```

当用户随后发送真实消息时，LLM 收到的消息序列为：
1. `[system prompt...]`
2. `[user] [The following is inherited context...]`
3. `[user] 用户的真实消息`

LLM 会将第 2 条识别为背景上下文，直接回复第 3 条。

    J --> K
    K --> L[写入 JSONL: {"role":"user","content":"[Inherited Context]..."}]
