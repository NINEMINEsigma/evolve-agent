# Evolve Agent — 开发规划与设计

一个以**运行时自我代码进化**为核心目标的 Agent。

---

## 设计

多个仓库完全隔离，任何进化失败都不会破坏当前运行实例, 
只维护一份持久化源码，所有运行时副本均由其派生，修改只在源仓库生效

---

## 运行时功能模块

### 网关与会话管理层

负责 WebSocket / HTTP 连接管理、用户会话生命周期维护、会话索引与归档。

- **会话创建与销毁**：每个用户连接对应一个独立会话，会话结束时归档历史
- **会话旋转**：当单一会话上下文超过容量限制时，自动归档旧会话并创建带有历史摘要的新会话，保持连续性
- **索引维护**：维护全局会话索引，支持按时间、标题、状态检索

### 消息处理循环

每个会话的核心处理单元，负责协调 LLM 调用、工具执行、上下文管理。

- **消息历史管理**：维护会话内的完整消息流，支持进程重启后从磁盘恢复
- **上下文压缩**：当总上下文接近模型窗口上限时，自动将较早历史压缩为摘要，保留近期完整细节
- **中断与取消**：支持用户主动中断当前处理中的请求，立即终止挂起的 LLM 调用和工具执行
- **工具事件流**：将工具调用与结果实时推送到前端，支持进度展示
- **LLM 流式推送**：LLM 响应通过 stream_delta / stream_done 消息逐块推送到前端，边生成边渲染，降低用户等待感

### LLM 客户端

对接大语言模型服务，屏蔽不同服务商的差异。

- **多模态兼容**：自动检测模型是否支持图像输入，不支持时降级为文本提示
- **Token 估算与追踪**：精确估算上下文占用，驱动压缩决策；追踪累计消耗
- **流式响应**：支持逐字返回，降低用户等待感
- **内容块错误处理**：当模型拒绝特定内容格式时，自动清洗并重试

### 工具系统

Agent 与外部世界交互的接口，分为多个层级：

- **核心工具**：文件系统操作、代码编辑、命令执行、前端构建、技能管理等
- **扩展工具**：网络搜索、网页抓取、文档处理、媒体处理、浏览器控制等
- **自定义工具**：用户可编写自己的工具脚本，启动时自动发现
- **外部工具桥接**：通过标准化协议对接外部工具服务

所有工具通过中央注册表管理，支持按名称分发、按危险等级审批、超时控制。

### 审批系统

对可能产生副作用的工具调用进行管控。

- **人工确认模式**：危险操作通过前端弹窗请求用户实时确认
- **自动审批模式**：部署本地轻量模型，对工具调用进行风险评估后自动通过或拒绝
- **审批上下文**：审批模型可看到与用户模型相同的系统提示和扩展上下文，确保判断一致

### 记忆系统

提供跨会话的长期记忆能力，采用双轨架构。

**会话级存储（审计轨）**：
- 保存完整的原始消息流，包括用户输入、模型输出、工具调用、推理过程等全部运行时细节
- 逐消息实时追加写入
- 用于进程重启后的状态恢复、上下文压缩决策、前端历史回放

**记忆级存储（回忆轨）**：
- 仅保留用户与助手的对话回合对，去除工具调用、推理过程等技术细节
- 回合结束后批量写入
- 用于跨会话关键字搜索、预取最近回合注入当前对话上下文、显式长期事实存储

### 模板系统

组装 System Prompt 的分层模板引擎，从上到下依次为：

1. **人格基因层**：最底层、不可变的人格定义
2. **角色灵魂层**：行为基调与核心价值观
3. **基础指令层**：通用约束与能力说明
4. **运行模式层**：根据当前状态（正常运行 / 修复模式）切换不同提示策略
5. **工具规范层**：工具使用规范与格式要求
6. **记忆指令层**：记忆系统注入的上下文使用说明
7. **技能层**：当前已加载技能的提示块

### 沙盒隔离

所有文件操作通过逻辑前缀命名空间隔离，禁止裸路径和目录遍历。

三个命名空间：
- **进化命名空间**：仅用于读写进化副本中的代码，运行态下可读写
- **工作命名空间**：通用 I/O 空间，存放用户文件、工具输出、技能文件等
- **修复命名空间**：仅在修复模式下可用，用于读取备份副本中的代码

## 扩展机制

系统提供多个热扩展点，无需修改核心源码：

- **自定义工具**：在特定目录下编写脚本并注册工具，启动时自动发现
- **本地模型**：在特定目录下放置模型文件，供审批系统或本地推理加载
- **自定义钩子**：编写脚本生成动态扩展上下文，插入到对话中作为实时信息源
- **技能文件**：以结构化文档形式定义技能，支持 frontmatter 元数据、动态加载与更新
- **预置技能模板**：内置面向自我进化的参考指南（架构设计、代码安全、调试策略等），Agent 可自行学习并转化为运行时技能
